## 项目简介

本项目是《人工智能编程语言》课程的期末大作业，使用 Python 实现了简单进化算法（Evolutionary Algorithm），用于解决数值优化问题。优化目标是最小化函数 min y = (x - 3)²，x ∈ [1, 4]，理论最优解为 x = 3，y = 0。

## 项目结构

项目包含以下文件：main.py 是主程序代码，包含完整的算法实现；requirements.txt 是项目依赖文件；README.md 是本说明文档；convergence_curve.png 是所有参数对比的收敛曲线图；.gitignore 是 Git 忽略文件配置。

## 功能说明

编码与解码方面，采用 9 位二进制编码表示浮点数 x，其中第 1 到 2 位表示整数部分（范围 0 到 3），第 3 到 6 位表示十分位（范围 0 到 9），第 7 到 9 位表示百分位（范围 0 到 9）。例如 x = 1.51 编码为二进制 101010001。解码验证显示原始 x 为 1.51，编码后为 101010001，解码后仍为 1.51，验证了编码解码的正确性。

交叉操作采用单点交叉，随机选择交叉点交换两个个体的编码信息。例如父代 1 为 101010001（对应 1.51），父代 2 为 101000001（对应 1.01），交叉后子代 1 为 101000001（对应 1.01），子代 2 为 101010001（对应 1.51）。

变异操作采用位翻转变异，每一位以概率 pm 进行 0 和 1 之间的翻转。例如原始个体 101010001（对应 1.51）变异后为 101010000（对应 1.50）。

选择操作采用锦标赛选择，k 值为 2，从种群中选择适应度较高的个体进入下一代。适应度函数定义为目标函数的倒数，即 fitness(x) = 1 / ((x - 3)² + 1e-6)，加 1e-6 是为了防止除零错误。

参数实验测试了 5 组不同的参数配置。第一组参数为 Gen=10, pc=0.6, pm=0.2, P0=10, P=15。第二组参数为 Gen=20, pc=0.6, pm=0.2, P0=10, P=15。第三组参数为 Gen=20, pc=0.6, pm=0.2, P0=20, P=30。第四组参数为 Gen=20, pc=0.6, pm=0.2, P0=5, P=7。第五组参数为 Gen=20, pc=0.2, pm=0.2, P0=10, P=15。

## 运行环境

运行本项目需要 Python 3.8 或更高版本，以及 matplotlib >= 3.3.0 和 numpy >= 1.19.0 两个依赖库。

## 快速开始

首先克隆项目到本地，执行 git clone https://github.com/1181413263/-.git 然后进入项目目录 cd AI_Prog_HW。接着安装依赖，执行 pip install -r requirements.txt，也可以使用国内镜像源加速安装。最后运行程序，执行 python main.py 即可。

## 运行结果

控制台输出首先进行编码解码验证，显示原始 x 为 1.51，编码后二进制为 101010001，解码后 x 为 1.51。验证多个值时，x=1.00 编码为 0100000000 解码为 1.00 误差为 0.0000，x=1.23 编码为 0100100011 解码为 1.23 误差为 0.0000，x=2.99 编码为 1010011001 解码为 2.99 误差为 0.0000，x=3.45 编码为 1101000101 解码为 3.45 误差为 0.0000。交叉操作验证显示父代 1 为 101010001 解码为 1.51，父代 2 为 101000001 解码为 1.01，子代 1 为 101000001 解码为 1.01，子代 2 为 101010001 解码为 1.51。变异操作验证显示原始个体为 101010001 解码为 1.51，变异个体为 101010000 解码为 1.50。

进化算法运行结果显示第一组参数 Gen=10, pc=0.6, pm=0.2, P0=10, P=15 的最优解为 x=2.9900，y=0.000100。第二组参数 Gen=20, pc=0.6, pm=0.2, P0=10, P=15 的最优解为 x=3.0000，y=0.000000。第三组参数 Gen=20, pc=0.6, pm=0.2, P0=20, P=30 的最优解为 x=3.0000，y=0.000000。第四组参数 Gen=20, pc=0.6, pm=0.2, P0=5, P=7 的最优解为 x=2.9800，y=0.000400。第五组参数 Gen=20, pc=0.2, pm=0.2, P0=10, P=15 的最优解为 x=2.9900，y=0.000100。

程序运行后会在项目文件夹中生成 convergence_curve.png 图片，展示所有参数设置下最优解的变化趋势对比。Param 1 对应 Gen=10, pc=0.6, P0=10，进化代数较少，收敛不够充分，最终解为 2.99。Param 2 对应 Gen=20, pc=0.6, P0=10，是基准配置，效果较好，收敛到最优解 3.00。Param 3 对应 Gen=20, pc=0.6, P0=20，种群更大，收敛更快更稳定，效果最佳。Param 4 对应 Gen=20, pc=0.6, P0=5，种群过小，容易陷入局部最优，效果最差。Param 5 对应 Gen=20, pc=0.2, P0=10，交叉概率低，搜索能力较弱，收敛较慢。

## 参数影响分析

进化代数 Gen 方面，代数越多搜索越充分，解的质量越高，Gen=10 时解的质量明显低于 Gen=20。交叉概率 pc 方面，较高的交叉概率（0.6）有助于探索解空间，pc=0.2 时收敛速度较慢，搜索能力不足。变异概率 pm 维持种群多样性，防止早熟收敛，本次实验固定为 0.2。种群规模 P0 方面，P0=20 时算法性能最佳，搜索更充分，P0=5 时算法稳定性差，易陷入局部最优。交叉后种群规模 P 与 P0 配合使用，较大的 P 值提供更多样本供选择，但计算成本相应增加。关于最优解一致性，多次运行同一组参数时，最优解在 2.98 到 3.02 之间波动，这是因为进化算法包含随机性，每次运行搜索路径不同，但所有结果都在理论最优解 x=3 附近，说明算法具有较好的鲁棒性。提高稳定性的方法包括增加进化代数 Gen、增大种群规模 P0 以及多次运行取平均值。

## 心得体会

算法方面，编码设计至关重要，编码方式直接影响搜索效率和精度，9 位二进制编码精度为 0.01，对于本问题足够，但对于高精度问题需要增加编码长度。参数调优是艺术，各参数相互耦合，没有万能参数组合，通过实验对比发现种群规模对性能影响最为显著。随机性影响结果，每次运行结果可能略有差异，体现了进化算法的随机搜索特性，这也是其能够跳出局部最优的原因之一。探索与开发的平衡方面，交叉操作负责探索，变异操作负责开发，两者需要平衡。

人机协作方面，AI 提升了编码效率，使用 Trae 和 DeepSeek 能快速生成算法框架代码，包括编码、解码、交叉、变异、选择等模块，节省了大量重复编码时间。AI 辅助调试方面，当运行结果出现异常如解码得到错误值时，AI 能快速指出编码位数和解码索引的潜在问题并给出修正建议。人类主导决策方面，AI 提供代码建议，但参数分析、结论推导和代码优化需要人类深度思考，例如 mutate 函数的实现，AI 最初使用了复杂的循环逻辑，需要手动优化。AI 是副驾驶，程序员负责定义问题、系统架构设计和最终决策，AI 负责生成模板代码和辅助调试，未来优秀程序员一定是善于向 AI 提问并驾驭其能力的 AI 增强型程序员。
